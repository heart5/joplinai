## 4. 关键发现

### 4.1 「jupyterhost」笔记本是性能黑洞

该笔记本包含大量 Jupyter/JupyterLab/JupyterHub 英文技术文档（直接保存的网页全文），特点：
- 单条笔记可达 **108 块**（如「Jupyter Notebook介绍、安装及使用教程」）
- 英文 token 效率虽高（~4 字符/token），但文本实在太长
- 自适应探测对每条笔记重复执行，造成大量冗余探测

### 4.2 探测缓存缺失

当前每条笔记独立执行自适应探测，即使同笔记本内文本特征相似。例如「时尚电玩」中同类长笔记的探测结果无法复用——每次都从 450 字符起二分搜索到 ~870 字符，单次成本 ~30-60s。

### 4.3 增强缓存全量冷启动

这是系统首次全量运行，`enhance_cache` 表为空。1,815 次 DeepSeek API 调用全部走网络，每条 ~1-2s。下次增量运行缓存命中率应大幅提升。
